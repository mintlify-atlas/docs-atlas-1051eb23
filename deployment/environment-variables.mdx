Redash is configured through environment variables. This reference lists all available configuration options extracted from the source code.

## Required Variables

These variables **must** be set for Redash to run:

### REDASH_COOKIE_SECRET

**Required**: Yes  
**Type**: String  
**Description**: Secret key for Flask session cookies and CSRF protection.

```bash
# Generate with:
openssl rand -base64 32
```

Without this variable, Redash will fail to start with an error.

### REDASH_SECRET_KEY

**Default**: Same as `REDASH_COOKIE_SECRET`  
**Type**: String  
**Description**: Secret key for encrypting data source options and sensitive configuration.

```bash
# Generate separately from cookie secret:
openssl rand -base64 32
```

## Database Configuration

### REDASH_DATABASE_URL

**Default**: `postgresql:///postgres`  
**Fallback**: `DATABASE_URL`  
**Type**: Database URL  
**Description**: PostgreSQL connection string for Redash's metadata database.

**Format**:
```
postgresql://[user[:password]@][host][:port][/dbname][?param=value]
```

**Examples**:
```bash
REDASH_DATABASE_URL="postgresql://redash:password@localhost/redash"
REDASH_DATABASE_URL="postgresql://user@postgres:5432/redash?sslmode=require"
```

### SQLALCHEMY_MAX_OVERFLOW

**Default**: `None`  
**Type**: Integer  
**Description**: Maximum number of connections to create beyond pool size.

### SQLALCHEMY_POOL_SIZE

**Default**: `None`  
**Type**: Integer  
**Description**: Number of connections to maintain in the pool.

### SQLALCHEMY_DISABLE_POOL

**Default**: `false`  
**Type**: Boolean  
**Description**: Disable connection pooling entirely.

### SQLALCHEMY_ENABLE_POOL_PRE_PING

**Default**: `false`  
**Type**: Boolean  
**Description**: Enable connection health checks before using from pool.

## Redis Configuration

### REDASH_REDIS_URL

**Default**: `redis://localhost:6379/0`  
**Fallback**: `REDIS_URL`  
**Type**: Redis URL  
**Description**: Redis connection string for caching and sessions.

**Examples**:
```bash
REDASH_REDIS_URL="redis://localhost:6379/0"
REDASH_REDIS_URL="redis://:password@redis:6379/0"
REDASH_REDIS_URL="rediss://redis:6380/0"  # SSL
```

### RQ_REDIS_URL

**Default**: Same as `REDASH_REDIS_URL`  
**Type**: Redis URL  
**Description**: Redis URL for RQ (job queue). Can be separate from main Redis.

## Web Server Configuration

### REDASH_HOST

**Default**: `""`  
**Type**: URL  
**Description**: Full URL where Redash is accessible (e.g., `https://redash.example.com`).

Used for generating absolute URLs in emails and external links.

### REDASH_WEB_WORKERS

**Default**: `4`  
**Type**: Integer  
**Description**: Number of Gunicorn worker processes for the web server.

**Recommended**: `(2 * CPU_CORES) + 1`

### REDASH_GUNICORN_TIMEOUT

**Default**: `60`  
**Type**: Integer (seconds)  
**Description**: Request timeout for Gunicorn workers.

### REDASH_GUNICORN_LIMIT_REQUEST_LINE

**Default**: `0` (unlimited)  
**Type**: Integer  
**Description**: Maximum size of HTTP request line in bytes.

### MAX_REQUESTS

**Default**: `1000`  
**Type**: Integer  
**Description**: Recycle workers after this many requests (prevents memory leaks).

### MAX_REQUESTS_JITTER

**Default**: `100`  
**Type**: Integer  
**Description**: Randomize worker recycling by ± this value.

## Worker Configuration

### WORKERS_COUNT

**Default**: `2`  
**Type**: Integer  
**Description**: Number of RQ worker processes to run.

### QUEUES

**Default**: All queues  
**Type**: String (comma-separated)  
**Description**: Specific RQ queues for workers to process.

**Available queues**:
- `queries` - Query execution
- `scheduled_queries` - Scheduled query execution
- `schemas` - Schema refresh
- `default` - Miscellaneous tasks

**Example**:
```bash
QUEUES="queries,scheduled_queries"
```

## Example Configuration

### Minimal Production Setup

```bash
# Required
REDASH_COOKIE_SECRET="<generate-32-char-secret>"
REDASH_SECRET_KEY="<generate-32-char-secret>"
REDASH_DATABASE_URL="postgresql://redash:password@postgres:5432/redash"
REDASH_REDIS_URL="redis://redis:6379/0"

# Application
REDASH_HOST="https://redash.example.com"

# Email
REDASH_MAIL_SERVER="smtp.example.com"
REDASH_MAIL_PORT="587"
REDASH_MAIL_USE_TLS="true"
REDASH_MAIL_USERNAME="redash@example.com"
REDASH_MAIL_PASSWORD="<smtp-password>"
REDASH_MAIL_DEFAULT_SENDER="redash@example.com"

# Security
REDASH_ENFORCE_HTTPS="true"
REDASH_ENFORCE_CSRF="true"
REDASH_COOKIES_SECURE="true"
```

### High-Performance Setup

```bash
# Web server
REDASH_WEB_WORKERS="8"
REDASH_GUNICORN_TIMEOUT="120"

# Workers
WORKERS_COUNT="8"
QUEUES="queries,scheduled_queries,schemas"

# Database pooling
SQLALCHEMY_POOL_SIZE="20"
SQLALCHEMY_MAX_OVERFLOW="10"
SQLALCHEMY_ENABLE_POOL_PRE_PING="true"

# Query execution
REDASH_ADHOC_QUERY_TIME_LIMIT="300"
REDASH_SCHEDULED_QUERY_TIME_LIMIT="600"
```
