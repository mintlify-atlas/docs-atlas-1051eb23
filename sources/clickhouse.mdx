## Overview

The ClickHouse data source enables you to connect to ClickHouse databases and run SQL queries. ClickHouse is a fast open-source column-oriented database management system optimized for real-time analytical queries.

## Connection Setup

To connect to ClickHouse, configure the following parameters:

### Required Fields

- **Database Name**: The name of the database to connect to

### Optional Fields

- **URL**: ClickHouse server URL (default: `http://127.0.0.1:8123`)
- **User**: Username for authentication (default: `default`)
- **Password**: Password for authentication (stored securely)
- **Request Timeout**: Query timeout in seconds (default: 30)
- **Verify SSL Certificate**: Whether to verify SSL certificates (default: true)

### Configuration Schema

```json
{
  "url": "http://127.0.0.1:8123",
  "user": "default",
  "password": "password",
  "dbname": "analytics",
  "timeout": 30,
  "verify": true
}
```

## Query Syntax

ClickHouse uses standard SQL syntax with extensions for analytical operations. Queries are executed directly without any JSON wrapper.

### Basic SELECT Query

```sql
SELECT 
  user_id,
  count() as visit_count,
  sum(duration) as total_duration
FROM visits
WHERE date >= '2024-01-01'
GROUP BY user_id
ORDER BY visit_count DESC
LIMIT 100
```

### Filtering with WHERE Clause

```sql
SELECT 
  event_name,
  count() as event_count,
  uniq(user_id) as unique_users
FROM events
WHERE 
  timestamp >= today() - INTERVAL 7 DAY
  AND event_type = 'click'
GROUP BY event_name
ORDER BY event_count DESC
```

### Date and Time Functions

```sql
SELECT 
  toDate(timestamp) as date,
  toHour(timestamp) as hour,
  count() as requests,
  avg(response_time) as avg_response_time
FROM logs
WHERE timestamp >= now() - INTERVAL 1 MONTH
GROUP BY date, hour
ORDER BY date DESC, hour DESC
```

### Window Functions

```sql
SELECT 
  user_id,
  purchase_date,
  amount,
  sum(amount) OVER (PARTITION BY user_id ORDER BY purchase_date) as cumulative_amount,
  row_number() OVER (PARTITION BY user_id ORDER BY purchase_date DESC) as purchase_rank
FROM purchases
WHERE purchase_date >= '2024-01-01'
ORDER BY user_id, purchase_date
```

## Aggregations

ClickHouse provides powerful aggregation functions optimized for analytical queries:

### Basic Aggregations

```sql
SELECT 
  product_category,
  count() as order_count,
  sum(quantity) as total_quantity,
  sum(price * quantity) as total_revenue,
  avg(price) as avg_price,
  min(price) as min_price,
  max(price) as max_price
FROM orders
WHERE order_date >= '2024-01-01'
GROUP BY product_category
ORDER BY total_revenue DESC
```

### Unique Count (Cardinality)

```sql
SELECT 
  toYYYYMM(event_date) as month,
  uniq(user_id) as unique_users,
  uniqExact(user_id) as exact_unique_users,
  count() as total_events
FROM user_events
WHERE event_date >= '2024-01-01'
GROUP BY month
ORDER BY month
```

### Percentiles and Quantiles

```sql
SELECT 
  service_name,
  quantile(0.5)(response_time) as p50,
  quantile(0.75)(response_time) as p75,
  quantile(0.90)(response_time) as p90,
  quantile(0.95)(response_time) as p95,
  quantile(0.99)(response_time) as p99
FROM service_metrics
WHERE timestamp >= now() - INTERVAL 1 DAY
GROUP BY service_name
ORDER BY p99 DESC
```

### Array Aggregations

```sql
SELECT 
  user_id,
  groupArray(product_id) as purchased_products,
  groupUniqArray(category) as unique_categories,
  arrayJoin(purchased_products) as product
FROM user_purchases
WHERE purchase_date >= '2024-01-01'
GROUP BY user_id
LIMIT 100
```

### Conditional Aggregations

```sql
SELECT 
  toDate(timestamp) as date,
  countIf(status = 200) as successful_requests,
  countIf(status >= 400 AND status < 500) as client_errors,
  countIf(status >= 500) as server_errors,
  sumIf(response_time, status = 200) / countIf(status = 200) as avg_successful_response_time
FROM http_logs
WHERE timestamp >= today() - INTERVAL 7 DAY
GROUP BY date
ORDER BY date DESC
```

### Group By with Rollup/Cube

```sql
SELECT 
  region,
  product_type,
  sum(sales) as total_sales
FROM sales_data
WHERE sale_date >= '2024-01-01'
GROUP BY ROLLUP(region, product_type)
ORDER BY region, product_type
```

## Advanced Features

### JOIN Operations

```sql
SELECT 
  u.user_id,
  u.username,
  count(o.order_id) as order_count,
  sum(o.total) as total_spent
FROM users u
LEFT JOIN orders o ON u.user_id = o.user_id
WHERE o.order_date >= '2024-01-01'
GROUP BY u.user_id, u.username
HAVING order_count > 5
ORDER BY total_spent DESC
LIMIT 50
```

### Subqueries and CTEs

```sql
WITH daily_stats AS (
  SELECT 
    toDate(timestamp) as date,
    count() as total_events,
    uniq(user_id) as unique_users
  FROM events
  WHERE timestamp >= today() - INTERVAL 30 DAY
  GROUP BY date
)
SELECT 
  date,
  total_events,
  unique_users,
  total_events / unique_users as events_per_user,
  avg(total_events) OVER (ORDER BY date ROWS BETWEEN 6 PRECEDING AND CURRENT ROW) as rolling_avg_events
FROM daily_stats
ORDER BY date DESC
```

### Array Operations

```sql
SELECT 
  user_id,
  tags,
  arrayFilter(x -> x != '', tags) as filtered_tags,
  arrayMap(x -> upper(x), tags) as upper_tags,
  has(tags, 'premium') as is_premium
FROM user_profiles
WHERE length(tags) > 0
LIMIT 100
```

### SAMPLE Clause (Data Sampling)

```sql
SELECT 
  event_type,
  count() as sampled_count,
  count() * 10 as estimated_total  -- Multiply by sampling ratio
FROM events SAMPLE 0.1  -- Sample 10% of data
WHERE event_date >= today() - INTERVAL 30 DAY
GROUP BY event_type
ORDER BY estimated_total DESC
```

### PREWHERE Optimization

```sql
SELECT 
  user_id,
  event_name,
  timestamp,
  properties
FROM events
PREWHERE toDate(timestamp) = today()  -- Evaluated first, reduces data read
WHERE event_name IN ('page_view', 'button_click')
LIMIT 1000
```

## Multiple Statements

You can execute multiple SQL statements in a single query by separating them with semicolons. The result of the last statement will be returned:

```sql
CREATE TEMPORARY TABLE temp_results AS
SELECT 
  user_id,
  sum(amount) as total_amount
FROM transactions
WHERE transaction_date >= today() - INTERVAL 7 DAY
GROUP BY user_id;

SELECT 
  t.user_id,
  u.username,
  t.total_amount,
  u.registration_date
FROM temp_results t
JOIN users u ON t.user_id = u.user_id
WHERE t.total_amount > 1000
ORDER BY t.total_amount DESC;
```

## Data Type Mapping

ClickHouse data types are mapped to Redash column types as follows:

- `Int8`, `Int16`, `Int32`, `Int64`, `UInt8`, `UInt16`, `UInt32`, `UInt64` → Integer
- `Float32`, `Float64`, `Decimal` → Float
- `Date` → Date
- `DateTime`, `DateTime64` → DateTime
- `String`, `FixedString`, `Enum`, `UUID` → String
- `Nullable(T)` → Same as type T

## Best Practices

1. **Use PREWHERE for filtering**: Place date/time filters in PREWHERE clause for better performance
2. **Limit result sets**: Always use LIMIT to avoid retrieving excessive data
3. **Leverage materialized views**: Create materialized views for frequently accessed aggregations
4. **Use appropriate data types**: Choose optimal data types to minimize storage and improve query speed
5. **Partition by date**: Partition tables by date for efficient time-range queries
6. **Use sampling for exploration**: Use SAMPLE clause when exploring large datasets
7. **Optimize JOINs**: Put smaller tables on the right side of JOINs
8. **Use dictionaries**: For dimension tables, use ClickHouse dictionaries for faster lookups
9. **Monitor query performance**: Use EXPLAIN and query_log to analyze query execution
10. **Set appropriate timeouts**: Configure timeouts based on expected query complexity
